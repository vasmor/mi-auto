# Правила вёрстки

**Назначение:** свод правил для HTML-разметки, CSS и JavaScript. Применимы к любому фронтенд-проекту.

---

## 1. Структура документа

### 1.1. Основные теги

- Использовать семантические теги: `header`, `main`, `footer`, `section`, `article`, `aside`, `nav`
- Внешняя обёртка страницы — `html`, внутри — `body`
- Основной контент — в `main`
- Навигация — в `nav`
- Хлебные крошки, логотип, меню — в `header` или отдельные блоки внутри него

### 1.2. Шапка (header)

- Корневой класс: блок шапки (например, `header`, `site-header`)
- Внутри: логотип, навигация, контакты — элементы блока (`header__logo`, `header__nav`, `header__contacts`)
- Обёртка для центрирования контента — контейнер (`header__container` или общий класс контейнера)

### 1.3. Подвал (footer)

- Корневой класс: блок подвала (например, `footer`, `site-footer`)
- Структура: колонки, ссылки, копирайт — элементы блока
- Контейнер — для ограничения ширины и выравнивания

### 1.4. Секции (section)

- Каждая логическая часть страницы — отдельная секция (`<section class="section-name">`)
- Внутри секции: контейнер, заголовок, контент — элементы
- Секции изолированы по стилям: классы секции не влияют на другие секции

### 1.5. Заголовки

- Иерархия: `h1` — один на страницу (обычно заголовок главного контента), далее `h2`, `h3`, `h4`
- Не пропускать уровни (`h2` → `h4` без `h3`)
- Заголовки секций — обычно `h2`
- Классы заголовков — элементы блока: `section-name__title`, `section-name__subtitle`

### 1.6. Контейнеры

- Общий контейнер страницы — ограничивает ширину, центрирует контент
- Внутри секций — собственный контейнер секции (`section-name__container`)
- Использовать `max-width` + `margin-inline: auto` для центрирования

## 2. БЭМ: блок, элемент, модификатор

### 2.1. Блок (block)

- Независимый компонент страницы. Имя — существительное, kebab-case
- Примеры: `header`, `card`, `form-section`, `social-links`
- Блок можно вкладывать в другой блок

### 2.2. Элемент (element)

- Часть блока, не имеет смысла вне блока. Имя — через **два подчёркивания**: `block__element`
- Примеры: `header__logo`, `card__title`, `form-section__input`
- Элемент всегда принадлежит одному блоку
- Элемент нельзя вкладывать в другой элемент: `block__elem1__elem2` — **неправильно**. Вместо этого — новый блок или плоская структура `block__elem1`, `block__elem2`

### 2.3. Модификатор (modifier)

- Вариант блока или элемента. Отдельный класс, начинается с **одного тире** `-`
- Модификатор **не** сливается с именем блока/элемента (`block__element -modifier`), не используется двойное тире (`block__element--modifier`)

**Правильно:**
- `card -compact`
- `form-section -home-1`
- `header__icon -pin`

**Неправильно:**
- `card--compact`
- `form-section__input--active`

### 2.4. Именование

- Все имена — kebab-case: `section-name`, `card-title`
- Блок и элемент: `block-name__element-name`
- Модификатор: ` -modifier-name` (с пробелом перед тире)

---

## 3. Одиночные (самозакрывающиеся) теги

Не использовать закрывающую косую черту `/` в одиночных тегах.

**Правильно:** `<img>`, `<input>`, `<br>`, `<meta>`, `<link>`  
**Неправильно:** `<img />`, `<input />`, `<br />`

---

## 4. Стили: архитектура

### 4.1. Базовый файл (base)

- Один общий файл с reset/normalize, типографикой, переменными
- Содержит: сброс отступов (`*`, `body`), стили заголовков, параграфов, ссылок, изображений
- CSS-переменные: шрифты, цвета, размеры, отступы — для переиспользования
- Подключается на всех страницах первым

### 4.2. Автономные стили секций

- Каждой секции — отдельный CSS-файл
- Стили секции **не зависят** от других секций: классы секции изолированы
- В файле секции — **только переопределения** относительно базового файла: не дублировать то, что уже задано в base
- Секция подключает свой файл только когда выводится на странице (условное подключение)

### 4.3. Критические стили

- Секции в пределах первых ~1200px от верха — подключать в critical CSS (в `<head>`) для быстрой отрисовки
- Остальные секции — в footer или отложенно

## 5. CSS: базовые принципы

### 5.1. Режим override-only

- Файлы секций пишут в режиме переопределения: только свойства, которые **отличаются** от base
- Не дублировать: `margin: 0`, `display: block`, `box-sizing: border-box` и т.п., если они уже в base
- Для изменения одного свойства не переписывать весь блок

### 5.2. Переменные

- Использовать переменные из базового файла: `--font-base`, `--color-text`, `--space-*`, `--fs-*`, `--radius-*`, `--lh-*`
- Не использовать «голые» значения, если есть переменная

**Ограничения по назначению:**
- `--space-*` — для любых свойств (отступы, размеры, gap)
- `--fs-*` — только для `font-size`
- `--radius-*` — только для `border-radius`
- `--lh-*` — только для `line-height`

Перед написанием CSS — проверить базовый файл на наличие переменных. `calc(...)` использовать только если подходящей переменной нет.

### 5.3. Порядок свойств

Соблюдать порядок «от внешнего к внутреннему»:

1. **Позиционирование:** `position`, `top`, `right`, `bottom`, `left`, `z-index`, `display`, `float`, `clear`
2. **Блочная модель:** `width`, `max-width`, `height`, `max-height`, `margin`, `padding`, `box-sizing`
3. **Границы:** `border`, `border-radius`
4. **Оформление:** `font`, `line-height`, `text-align`, `color`, `background`, `box-shadow`
5. **Разное:** `animation`, `cursor`, `opacity`, `transform`, `transition`

**Уточнения:**
- После `display: flex` — сразу flex-свойства (`flex-direction`, `align-items`, `justify-content`, `gap`, `flex-wrap`)
- После `display: grid` — сразу grid-свойства
- `max-width` сразу после `width`, `max-height` сразу после `height`

### 5.4. line-height

Использовать безразмерные значения (`1`, `1.17`) или переменные. Не писать `calc(... * 1em)` или значения с единицами.

### 5.5. Общие принципы

- Избегать избыточной специфичности
- Минимизировать вложенность селекторов
- Не использовать `!important`, кроме крайней необходимости
- Использовать классы, а не id, для стилизации

---

## 6. CSS Grid: единица 1fr

`1fr` использовать только в двух случаях:
1. Равные колонки/строки: `grid-template-columns: 1fr 1fr`
2. «Остаток» после фиксированных размеров: `grid-template-columns: 200px 1fr`

В остальных случаях — конкретные размеры из макета (vw, calc, px и т.д.).

---

## 7. Кнопки и компоненты

- Общая система кнопок — базовые стили и переменные
- Для вариантов — переопределять только нужные переменные
- Не переписывать базовую систему для локального удобства
- Теги/чипы: при отсутствии отдельного hover в макете — применять hover как у активного состояния

---

## 8. Скрипты: архитектура

### 8.1. Базовый скрипт

- Общий скрипт: меню, модальные окна, общие обработчики
- Подключается на всех страницах

### 8.2. Автономные скрипты секций

- Каждой секции с интерактивом — отдельный JS-файл
- Скрипт секции проверяет наличие соответствующего элемента на странице; при отсутствии — выходит без инициализации
- Секция не зависит от порядка подключения других секций
- Подключать скрипт секции **только** когда секция выведена на странице

### 8.3. Структура скрипта секции

- IIFE или модуль для изоляции
- В начале: поиск корневого элемента секции по селектору; если не найден — `return`
- Инициализация внутри обработчика `DOMContentLoaded` (или сразу, если DOM уже готов)
- Не создавать глобальные переменные без необходимости; при необходимости — namespace

### 8.4. Общие библиотеки (Swiper, Fancybox и т.д.)

- Подключать библиотеку один раз
- Использовать менеджер ресурсов (lazy loading) для секций ниже первого экрана
- Секции регистрируют инициализацию; менеджер загружает библиотеку и вызывает зарегистрированные колбэки

## 9. Figma и разработка по макету

### 9.1. Источники данных

- Все значения CSS — только из Figma API или расчётного документа (EXACT_CALCULATIONS)
- Запрещено придумывать, округлять или «подгонять» значения
- При недоступности обязательного источника — остановить выполнение и сообщить

### 9.2. Единицы измерения

Вёрстка должна быть **резиновой**: привязанной к ширине экрана и пропорционально масштабирующейся при увеличении и уменьшении окна браузера.

- Desktop: `calc(VALUE / var(--vw-base) * 100vw)` (VALUE — из Figma API)
- Mobile: `calc(VALUE / var(--vw-base) * 100vw)`

Переменная `--vw-base` — ширина макета в Figma для десктопной и мобильной версии. Задаётся в корневом элементе:

```css
:root {
    --vw-base: 1440;
}
@media (max-width: 767px) {
    :root {
        --vw-base: 370;
    }
}
```

### 9.3. Раскладка и порядок

- Порядок и layout брать из Figma API (`children`, `absoluteBoundingBox`)
- Рекурсивно обрабатывать все элементы
- Не выводить порядок «по смыслу» без сверки с API

### 9.4. Запрещённые режимы

- «Сделать похоже», visual hotfix
- Визуальные правки без расчётов по макету
- Округление или предположения вместо точных значений
